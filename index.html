windows 安装MySQL压缩包

安装MySQL

  1. 下载MySQL

  2. 解压MySQL并且进入MySQL的bin目标

  3. 执行: mysqld —explicit_defaults_timestamp=true #初始化数据库

  4. 执行: mysqld —console #启动数据库服务

  5. 执行: mysql -u root mysql #启动数据库
    执行: use mysql
    执行: update user set password= PASSWORD(“设置密码”) where user=”root” #设置管理员密码
    执行: n #立即刷新

  6. 注册MySQL服务到Windows
    执行: mysqld — install MySQL #最后一个参数是服务名, 可随意定义

  7. 为了不用每次都切换目录使用MySQL, 将MySQL的bin目录路径添加到环境变量

注意: 如果Windows无法启动MySQL的服务, 请检查注册表: HKEY_LOCAL_MACHINE-SYSTEM-CurrentControlSet-services-mysql右边的ImagePath属性, 看改值是否指向MySQL解压缩文件的路径, 如果不是请修改

汇编除法运算

汇编的除法指令是”div”, 实际上相对高级语言的除法, 汇编指令的div是同时得到商和余数的, 结果分别保存在不同的寄存器.

  1. 除数: 有8位和16位两种, 在一个寄存器或者内存单元中
  2. 被除数: 如果除数是8位则被除数为16位, 默认放在AX中, 如果除数是16位, 则被除数为32位, 默认高位放在DX, 低位放在AX
  3. 结果: 如果除数是8位, 那么执行div后, 余数存放在AH, 商存放在AL中; 如果除数是16位, 那么AX保存商, DX保存余数
    注意: 被除数的位数一定是除数的两倍

指令格式:
1: div registre
2: div 内存单元

一个除数为8位的例子(96除以10)

mov ax 60h ;dec 96 = hex 60
mov bh 0ah ;dec 10 = hex ah, 代码中数字起始位位字母要在最前加0
div bh ;执行后ax为:0609, 即高位ah=06(余数), 低位al=09(商)

执行div后ax的高低位就可以后后续的计算了

使用位运算转换大小写

大小写字母的ASCII码差值为32, 32是2的5次方, 因此如果将ASCII的值使用二进制编码表示会不会有奇特的关系呢?
看看如下列表
ch dec bin ch dec bin
a 97 0110 0001 -> A 65 0100 0010
b 98 0110 0010 -> B 66 0100 0010
c 99 0110 0011 -> C 67 0100 0011
d 100 0110 0100 -> D 68 0100 0100
e 101 0110 0101 -> E 69 0100 0101
从列表可以看出, 二进制ASCII码的低四位是一致的, 唯一的差别就在于第5位(从0开始数, 2的5次方正好是32)
小写字母的是1, 大写字母的是0,

于是我们在编程中可以使用为运算直接将这一位置为1或者置为0实现大小写的转换, 所以, 将大小字母转换为小写字母即保留其他位, 使用”或0010 0000”(代码即”!32”)实现转换, 同样, 如果要将第小写转为大小, 用”与1101 1111”(代码即”&-33”)把5位设为0即可.

以下给的c++使用以上方法转换的代码片段
char ch = ‘A’;//定义ch的的值为’A’
ch != 32;//将ch转换为小写’a’
ch &= -33;//将小写的’a’转换为’A’
